BidEval es una plataforma de inteligencia artificial que automatiza la evaluación de propuestas de proveedores para proyectos de infraestructura energética. Propuesta de valor: Reducir el tiempo de evaluación de ofertas en un 60%, garantizando consistencia, trazabilidad y total privacidad de datos mediante procesamiento 100% local.

2. El Reto del Procurement en Energía

En proyectos de infraestructura energética —plantas de hidrógeno verde, parques solares, instalaciones BESS— el proceso de selección de proveedores representa uno de los cuellos de botella más críticos del ciclo de proyecto.

El Proceso Actual: Lento, Costoso y Propenso a Errores

Tiempo Perdido

  • Evaluar 5-7 proveedores para un proyecto típico requiere 2-4 semanas de trabajo dedicado
  • Cada propuesta contiene 50-200 páginas de documentación técnica y económica
  • La comparación manual de requisitos consume +100 horas de ingenieros senior

Costos Ocultos

  • Horas de personal técnico cualificado (€80-150/hora)
  • Retrasos en cronogramas de proyecto
  • Oportunidades perdidas por decisiones tardías

Riesgos de Calidad

  • Inconsistencias en la evaluación entre proveedores
  • Subjetividad en la interpretación de requisitos
  • Errores humanos por fatiga en documentos extensos
  • Falta de trazabilidad en decisiones

Estadísticas del Mercado

  • El 67% de los profesionales de procurement consideran que el proceso de evaluación es ineficiente
  • El 45% reporta haber tomado decisiones basadas en información incompleta
  • El costo promedio de una mala selección de proveedor supera el 15% del valor del contrato

Impacto Económico

Concepto Impacto Típico
Horas de evaluación manual 120+ horas por proyecto
Costo de personal técnico €12,000 - €18,000 por evaluación
Retrasos por proceso lento 2-4 semanas adicionales
Riesgo de mala selección 15% del valor del contrato

4. Profundización Técnica

Arquitectura del Sistema

Sistema de Scoring Multicriteria

BidEval implementa un sistema de puntuación con 12 criterios organizados en 4 categorías principales, basado en mejores prácticas del sector energético:

Categoría Peso Criterios Incluidos
TÉCNICA 35% Eficiencia BOP (12%), Vida útil (8%), Flexibilidad (8%), Pureza/Presión (7%)
ECONÓMICA 35% CAPEX total (18%), OPEX garantizado (12%), Garantías (5%)
EJECUCIÓN 12% Plazo entrega (6%), Track record (3%), Solidez proveedor (3%)
HSE/ESG 18% Seguridad ATEX (12%), Sostenibilidad (6%)
Personalización: Todos los pesos son completamente configurables. Los valores mostrados corresponden al proyecto H2 La Zaida (hidrógeno verde) y pueden adaptarse a cada vertical o proyecto específico.

Tecnología RAG (Retrieval-Augmented Generation)

BidEval utiliza tecnología RAG para evaluar propuestas contra requisitos:

  1. Vectorización: Cada documento se convierte en embeddings semánticos
  2. Búsqueda: Para cada requisito, se recuperan los fragmentos más relevantes
  3. Evaluación: La IA analiza el contexto y determina el nivel de cumplimiento
  4. Clasificación: INCLUIDO, PARCIALMENTE INCLUIDO, NO INCLUIDO, NO APLICA

7. Seguridad y Compliance

Principio Fundamental

Los datos nunca abandonan el entorno del cliente. Todo el procesamiento de IA se realiza localmente, sin conexión a APIs cloud externas.

Arquitectura de Seguridad

Aspecto Implementación
Procesamiento de IA 100% local, sin llamadas a APIs cloud
Almacenamiento de datos Base de datos on-premise en servidor del cliente
Transmisión de datos Solo red interna, sin tráfico externo
Control de backups 100% del cliente
Autenticación Sistema local, integrable con SSO corporativo

Ventaja Competitiva: IA Local vs Cloud

Aspecto BidEval (Local) Competencia (Cloud)
Datos de licitaciones Nunca salen del servidor Enviados a terceros
Riesgo de filtración Cero Dependiente del proveedor
Cumplimiento normativo Simplificado Requiere evaluación de terceros
Auditoría Control total Limitada a logs de proveedor

Cumplimiento Normativo

Compatible con:

  • GDPR (Protección de datos UE)

  • Requisitos de confidencialidad en licitaciones
  • Normativas de infraestructura crítica

Garantías:

  • Cero datos transmitidos externamente
  • Cero dependencia de APIs cloud
  • Control total sobre modelos y configuraciones
  • Logs de auditoría completos

8. Guía de Implementación

Proceso de Adopción

Fases de Implementación

Fase 1: Kick-off y Diseño (Semana 1)

  • Reunión de inicio con stakeholders
  • Definición de criterios de scoring específicos
  • Identificación de flujos de trabajo actuales
  • Planificación de infraestructura

Fase 2: Instalación (Semana 2)

  • Despliegue de servidor de aplicación
  • Instalación de modelos de IA locales
  • Configuración de base de datos
  • Pruebas de conectividad y rendimiento

Fase 3: Configuración (Semana 3)

  • Personalización de criterios de scoring
  • Configuración de pesos por categoría
  • Integración con sistemas existentes (opcional)
  • Pruebas con documentos de ejemplo

Fase 4: Formación (Semana 4)

  • Capacitación de usuarios clave (hasta 10 personas)
  • Documentación de procedimientos
  • Prueba piloto con proyecto real
  • Ajustes finales basados en feedback
